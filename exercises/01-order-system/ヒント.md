# ヒント: マジックナンバーの使用

## 問題点

このコードには以下の問題があります：

1. **数値の意味が不明瞭**
   - `status === 0`, `status === 1` などの数値が何を意味するのか分からない
   - `0.1`, `0.05` などの割合の意味が不明
   - `100`, `1000000` などの閾値の意味が分からない

2. **保守性の低下**
   - 数値の意味を変更したい場合、コード全体を検索して修正する必要がある
   - 同じ意味の数値が複数箇所に散らばっている可能性がある

3. **可読性の低下**
   - コードを読む人が数値の意味を推測する必要がある

## 改善のポイント

1. **定数の導入**
   - 意味のある名前を持つ定数を定義する
   - TypeScriptの`const`や`enum`を活用する

2. **定数の配置**
   - 関連する定数をグループ化する
   - 定数の命名規則を統一する（UPPER_SNAKE_CASEなど）

3. **型安全性の向上**
   - `enum`を使用することで、型チェックの恩恵を受ける
   - 不正な値の代入を防ぐ
