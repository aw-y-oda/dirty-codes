# TypeScript リファクタリング演習 🔧

ようこそ！このディレクトリには、実践的なリファクタリング演習問題が用意されています。

## この演習の目的

- **コードの問題を見抜く力**を養う
- **リファクタリングの手法**を実践的に学ぶ
- **保守しやすいコード**の書き方を身につける
- TypeScriptでの設計パターンに慣れる

## 演習の進め方

### 1. 問題を読む
各ディレクトリにある`問題.ts`を開いて、コードを読んでみてください。
一見動きそうなコードですが、実は改善の余地がたくさんあります。

### 2. 問題点を探す
「このコード、なんか変だな」と思うところを探してみましょう。
- 読みにくい部分はありませんか？
- テストしづらそうな部分は？
- 後から変更しづらそうな部分は？

### 3. 考えてリファクタリング
まずは自分なりに考えて、リファクタリングしてみてください。
正解は一つではありません。あなたなりのアプローチでOKです。

### 4. ヒントを見る（必要に応じて）
詰まったら`ヒント.md`を見てみましょう。
どんな問題があるか、どうアプローチすればいいかのヒントが書いてあります。

### 5. 回答例を参考にする
自分なりのリファクタリングができたら、`回答例.md`を見てみましょう。
いろいろなアプローチがあることに気づくはずです。

## 重要なポイント

### 🎯 答えは一つじゃない
リファクタリングには「これが唯一の正解」というものはありません。
状況やチームの方針によって、ベストなアプローチは変わります。

### 💪 失敗してもOK
最初からうまくリファクタリングできなくても大丈夫。
試行錯誤しながら学ぶのが一番の成長につながります。

### 🤝 相談しながら進めよう
わからないことがあったら、周りの人と相談しながら進めてください。
「こっちのやり方とこっちのやり方、どっちがいいと思う？」みたいな
議論が一番勉強になります。

### 🎈 気軽にやってみよう
完璧を目指さなくてOK。まずは気軽に問題を読んで、
「ここ変だな」と思うところを見つけるところから始めてみてください。

## ディレクトリ構成

```
XX-domain-name/
├── 問題.ts       ← まずはここから読んでみよう
├── ヒント.md     ← 困ったら見てみよう
└── 回答例.md     ← 自分でやってから見よう
```

## Tips

- **焦らず一つずつ**: 問題が複数見つかっても、一つずつ順番に改善していきましょう
- **動作確認しながら**: リファクタリング後も同じように動くか確認しながら進めましょう
- **コミット細かく**: 小さな変更ごとにコミットすると、後から見返しやすいです
- **楽しむ**: コードをきれいにしていく過程を楽しみましょう！
